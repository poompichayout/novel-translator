# Week 4 — 4-Channel RAG + End-to-End Translation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the 4-channel RAG context builder, plug it into the translation worker, run one novel × 10 chapters end-to-end, and emit aligned pairs to `train.jsonl`.

**Architecture:** Context builder runs four parallel queries (entity-first SQL, hybrid BM25+vector, recent plot summary, glossary dictionary), merges and ranks results, and renders into a stable prompt-cache-friendly prefix. Worker passes the prefix unchanged across chapters of the same novel to maximize cache hits.

**Tech Stack:** Go (queries), pgvector (vector channel), Postgres `tsvector` + `ts_rank` (BM25-ish channel), Gemini `text-embedding-004` (existing alignment), Anthropic Sonnet 4.6.

**Source spec:** `docs/phase-1-spec.md` Component 2 (RAG) and Week 4 build-order row.

---

## Scope

- New package `internal/rag`:
  - `Channel` interface: `Fetch(ctx, query) ([]Snippet, error)`.
  - `EntityChannel` — SQL lookup by entity name match in source sentence.
  - `HybridChannel` — `tsvector` + pgvector fusion (weighted sum or RRF).
  - `RecentPlotChannel` — last 3 chapter summaries (new `chapter_summaries` table; summaries generated by an extra Sonnet call cached separately).
  - `GlossaryChannel` — flat dictionary lookup.
  - `ContextBuilder` — calls all four in parallel, merges, returns prompt prefix string.
- Migration `005_add_chapter_summaries.up.sql`:
  - `chapter_summaries(chapter_id PK, summary_en TEXT, summary_th TEXT, created_at)`
- Wire builder into `SonnetAdapter` request.
- Run `aligner.py` on the 10 translated chapters → confirm `train.jsonl` populates.

## Out of Scope

- Apache AGE / graph traversal (explicitly out of Phase 1).
- Microsoft GraphRAG.
- Auto-summary scheduling (manual trigger Phase 1).

## Acceptance

- One novel, 10 chapters: pasted → translated end-to-end → aligned → `train.jsonl` non-empty with valid JSONL pairs.
- Entity name appears consistently across all 10 translated chapters (manual sanity check).
- Prompt cache reads `cached_input_tokens / input_tokens > 0.5` average across the 10 chapters.

## Notes

- Stable cache prefix order matters: `[system → static glossary → recent plot → entity hits → BM25/vector]`. Anything that changes per-chapter must come after the cache point.
- Phase 1 success criteria #1–#4 should be ticked after this week.
